1. Чтобы перейти в директорию, используется команда "cd" с указанием пути к желаемой директории. 

2. "~" - это символ для обозначения домашней директории пользователя.
"." - это символ для обозначения текущей директории.
".." - это символ для обозначения родительской директории текущей директории.

3. "rm" - это команда для удаления файлов и папок в командной строке.

4. "cd" - это команда для изменения текущей директории.

5. "cp" - это команда для копирования файлов и папок.

6. "mv" - это команда для перемещения или переименования файлов и папок.

7. Для переименования папки или файла используется команда "mv" с указанием старого имени и нового имени. 

8. Чтобы переместить файл или папку используется команда "mv" с указанием пути к исходному файлу/папке и пути к новому месту назначения.

9. Флаг "-R" (или "--recursive") используется совместно с командами "cp" и "mv" для копирования или перемещения файлов/папок рекурсивно, т.е. включая все поддиректории и их содержимое.

10. Флаг "-r" (или "--rename") используется с командой rm для рекурсивного удаления файлов и папок внутри указанной директории. 

11. Git - это система контроля версий, которая позволяет отслеживать изменения в коде и управлять историей проекта.

12. "git init" - это команда, используемая для инициализации нового репозитория в пустой директории. Когда вы выполняете "git init", Git создает новую поддиректорию ".git", в которой хранится всё необходимое для работы с репозиторием.

13. Чтобы создать новую ветку в Git, можно использовать команду "git branch" с указанием имени новой ветки.

14. Чтобы переключиться на другую ветку в Git, можно использовать команду "git checkout" с указанием имени целевой ветки.

15. Слияние (merge) в Git - это процесс объединения изменений из одной ветки с другой.

16. "git rm" - это команда, используемая для удаления файлов из Git-репозитория.

17. "git revert" - это команда, используемая для отмены предыдущего коммита путем создания нового коммита, который отменяет изменения в предыдущем коммите. Это позволяет сохранить историю изменений и делает отмену коммита безопасной для существующих веток.
